Manuskript exportieren ...

Manuskript exportieren: Eine Gesamtdatei wird erstellt und heruntergeladen. Dieser Vorgang kann einen Moment dauern. Dabei werden alle Texte der Einzeldateien vereinigt und im <sourceDoc> in die entsprechenden <surface> kopiert. Diese Datei kann dann in unsere TeiPublisher-App Nietzsche Druckmanuskripte importiert werden.

Datei exportieren

Ausgewählte Datei exportieren: Die im Dialog ausgewählte Einzeldatei wird zur Voransicht in unserer TeiPublisher-App Nietzsche Druckmanuskripte heruntergeladen. Damit können auch die verschiedenen nicht-topologischen Aspekte der Daten in unserer TeiPublisher-App angezeigt werden.
Zur Kennzeichnung endet der Dateiname mit "_tp.xml", also z.B. "D20_a1r_GMTitelseite.xml" wird beim Download zu "D20_a14_GMTitelseite_tp.xml". Diese Datei enthält im <sourceDoc> auch Text. Die Datei kann aber wieder in topoTEI importiert werden. Dabei wird der Text im <sourceDoc> entfernt und die Kennzeichnung "_tp.xml" wird aus dem Dateinamen ausgeschnitten.

Alle Dateien exportieren

Alle Einzeldateien als ZIP exportieren: Alle Einzeldateien werden ohne Text im <sourceDoc> als ZIP-Datei heruntergeladen. Dies kann zur Sicherung der Dateien verwendet werden.

Textblock

Aus den <div2> von <text> werden im <surface> des <sourceDoc> Textblöcke generiert. Bei diesen Textblöcken könne oben und unten jeweils Abstände eingefügt werden. Der Textblock am Anfang der Seite hat per Default 5em padding-top. Dies entspricht N's Leerzeilen am Anfang jeder Seite. Der Textblock unten hat ebenfalls 5em padding-bottom.
Der Textblock-Dialog (Settings für Textblock) wird per Klick auf die Zeilennummer einer regulären Zeile (blaue Zeilennummer) aufgerufen. Neben padding-top und padding-bottom des aktuellen Textblocks kann auch die Zeilenhöhe aller regulärer Zeilen eingestellt werden.

Positionierbare Elemente

Positionierbare Zeilen

Positionierbare Zeilen erscheinen mit grauer Zeilennummer. Per Klick auf die Zeilennummer bzw. beim Verschieben der entsprechenden Zeilenelemente erscheint der Dialog Position für Zeile {@n}:

  • top: legt die relative Top-Position gegenüber des dazugehörenden Textblocks fest (in em)
  • margin-left: Abstand nach links in em
  • z-index: Der z-index bestimmt, ob ein Element über oder unter einem anderen Element liegt, wenn sie sich aufgrund ihrer Koordinaten überlagern. Elemente mit grösserem z-index sind über Elementen mit tieferem z-index. Dieser Wert kann verändert werden (auf 1 setzen), um allenfalls Elemente auszuwählen, die aufgrund ihrer Position nicht ausgewählt werden können.
Ausserdem erscheint wie für alle positionierbaren Elemente der Dialog Positionen

Add

Add mit dem Attribute "above" oder "below" sind positionierbar. Diese Elemente werden relativ zu ihrem Einfügungsort positioniert. Sie können entweder per Mouse (drag+drop) oder nach ihrer Auswahl per Mouseclick über die Pfeiltasten bewegt werden. Bei der Tastaturbewegung kann die Distanz über die Modifikationstasten beeinflusst werden (z.B. Ctrl+Up). Wird Ctrl bereits während des Auswahlclick gedrückt, dann wird das Element um ein bestimmte Distanz verschoben. Bei erneutem Ctr+Mouseclick springt das Element auf den Ausgangspunkt zurück. Mehrfachauswahl erfolgt mittels Shift+Mouseclick.
Die Positionen aller Adds auf der Zeile eines ausgewählten Add werden im Dialog Positionen angezeigt.

FW

FW werden wie Positionierbare Zeilen relative zum Textblock positioniert. Ihre Positionen werden im Dialog Positionen angezeigt.

Notes

Notes werden wie Positionierbare Zeilen relative zum Textblock positioniert. Ihre Positionen werden im Dialog Positionen angezeigt.

Head

Head sowie <seg[@type='head'/> lassen sich horizontal positionieren. Ihre Positionen werden im Dialog Positionen angezeigt.

flushRight

Der Inhalt von Zeilen mit dem Attribut @rend="flushRight" lassen sich horizontal positionieren. Ihre Positionen werden im Dialog Positionen angezeigt.

Positionen

Der Positionen Dialog erscheint, wenn ein positionierbares Element ausgewählt ist. Der Dialog kann per drag'n'drop innerhalb des Browsers positioniert werden.

  • keyListener: Der keyListener kann ein oder ausgeschaltet werden. Der keyListener hört auf alle Tastatureingaben. Will man eine Zelle der Tabelle manuell bearbeiten, muss der keyListener ausgeschaltet werden, damit er die Angabe nicht abfängt. Sollte der Editor nicht mehr per Tastatur funktionieren, so kann es sein, dass der keyListener verloren ging. Er kann dann über diesen Dialog wieder eingeschaltet werden.
  • absolute/relative Positionsdaten: Absolute Positionsdaten zeigen, die Position der Elemente auf dem Bildschirm. Relative dagegen die relativen Werte (siehe unter Positionierbare Elemente was jeweils die Referenz für die relative Position darstellt). Alle absoluten Werte werden bei der Positionierung in relative Werte umgewandelt. Deshalb kann es vorkommen, dass die manuelle Eingabe nicht korrekt übernommen wird. Hier hilft jeweils eine wiederholte Eingabe bis der gewünschte Wert erreicht ist.
  • Angleichung der Werte: Soll ein Wert für mehrere Element gesetzt werden, dann kann der Wert per Doppelclick auf die entsprechende Zeile ausgewählt werden. Unter der Tabelle erscheint nun dieser Wert in einem eigenen Eingabefeld, davor das ensprechende Label. Der Wert kann hier angeglichen werden (der Click ins Feld setzt automatisch den keyListener aus). Mittels des Buttons wird der Wert bei allen ausgewählten Elementen gesetzt (dabei wird der keyListener wird aktiviert). Aufgrund der Umwandlung von absoluten Werten zu relativen, muss der Button jeweils mehrfach betätigt werden, bis das gewünschte Resultat erreicht ist.
  • z-index: für jedes Element kann ein z-index in der letzten Spalte der Tabelle gesetzt werden. Der z-index bestimmt, ob ein Element über oder unter einem anderen Element liegt, wenn sie sich aufgrund ihrer Koordinaten überlagern. Elemente mit grösserem z-index sind über Elementen mit tieferem z-index. Dieser Wert kann verändert werden (auf 1 setzen), um allenfalls Elemente auszuwählen, die aufgrund ihrer Position nicht ausgewählt werden können.

Layout

Das Layout der Seite kann durch mehrere Faktoren beeinflusst werden:

  • Minimale Seitendimensionen: siehe Seite.
  • Zeilenabstände oben, unten sowie zwischen Abschnitten über das padding-top und padding-bottom der Textblöcke.
  • Leerzeilen: In der TEI/XML-Datei können Leerzeilen eingefügt werden. Ein Leerzeile ist ein <lb/> unmittelbar vor einem anderen <lb/>. Diese werden ins <sourceDoc> kopiert und haben die gleiche Zeilenhöhe wie alle regulären Zeilen.
    Wichtig: Das <lb/> soll keine Zeilennummer haben und muss direkt vor einem darauffolgenden <lb/> stehen. Es darf weder in einem vorangehenden übergeordneten Element wie <div2/> oder <p/> stehen, das das darauffolgende <lb/> nicht enthält, noch darf das darauffolgende <lb/> in einem übergeordneten Element stehen, das das erste nicht enthält.
